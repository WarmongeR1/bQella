\label{classraEdit_a88d6a5ed8345f496dfa43a78a9240d1a}
This function is called when the document is loaded. The {\itshape type\/} indicates the type of resource to be loaded. For each image in the document, this function is called once.

The default implementation ignores {\itshape type\/} and tries to locate the resources by interpreting {\itshape name\/} as a file name. If it is not an absolute path it tries to find the file in the paths of the  searchPaths property and in the same directory as the current source. On success, the result is a QVariant that stores a QByteArray with the contents of the file.

If you reimplement this function, you can return other QVariant types. The table below shows which variant types are supported depending on the resource type:

the name of the displayed document. 

This is a an invalid url if no document is displayed or if the source is unknown.

When setting this property \hyperlink{classraEdit}{raEdit} tries to find a document with the specified name in the paths of the searchPaths property and directory of the current source, unless the value is an absolute file path. It also checks for optional anchors and scrolls the document accordingly

!! -\/$>$ If the first tag in the document is {\ttfamily }\{$<$qt type=\char`\"{}detail\char`\"{}$>$\}, the document is displayed as a popup rather than as new document in the browser window itself. Otherwise, the document is displayed normally in the text browser with the text set to the contents of the named document with setHtml().
